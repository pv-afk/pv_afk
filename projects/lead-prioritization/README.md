# Приоритизация обращений

В этом проекте я решаю задачу ранжирования обращений: для каждого обращения оцениваю вероятность успешного целевого действия в течение пяти дней после назначения.

Основная метрика — **Daily Average Precision**. Сначала Average Precision считается отдельно для каждого дня назначения, затем результаты усредняются по дням.

## Результат

Я сравнил итоговый ансамбль с базовой версией на двух последовательных временных окнах внутри обучающей выборки:

| Временное окно | Baseline | Версия v2 | Изменение |
|---|---:|---:|---:|
| 13–17 апреля | 0.714285 | 0.738610 | +0.024325 |
| 18–22 апреля | 0.740219 | **0.760062** | +0.019843 |

Результат `0.760062` получен на локальном временном holdout. Скрытая тестовая выборка платформы недоступна, поэтому её точное значение заранее неизвестно.

## Что я сделал

- Использовал только признаки и события, доступные до момента назначения: `event_ts < assignment_ts`.
- Добавил давность последних событий по типам, источникам и контекстам.
- Собрал дополнительные окна активности: 6 и 12 часов, 2, 5, 10, 21 и 60 дней.
- Добавил признаки последних действий пользователя и переходов между ними.
- Восстановил округлённые версии накопительных счётчиков и величину остатка после округления.
- Проверил решение на двух непересекающихся временных holdout.
- Объединил исходный ансамбль, два варианта LightGBM и CatBoost.

Финальные веса ансамбля:

- 20% — исходный ансамбль;
- 30% — LightGBM на расширенных признаках;
- 20% — Extra-Trees LightGBM;
- 30% — CatBoost.

## Файлы

- `lead_prioritization_solution_v2.ipynb` — основной notebook с описанием подхода и кодом;
- `solution_v2.py` — запуск улучшенного решения;
- `solution.py` — функции исходного ансамбля, которые использует версия v2;
- `submission_v2.csv` — готовый файл предсказаний;
- `requirements.txt` — зависимости проекта.

Исходные `train.csv`, `test.csv` и `events.csv` не публикуются. Для запуска их нужно положить в каталог `data`.

## Запуск

```bash
python -m pip install -r requirements.txt
python solution_v2.py --data-dir data --output submission_v2.csv
```

Файл результата содержит ровно две колонки: `lead_id` и `score`.
